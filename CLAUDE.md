## What This Project Is

LeadHarvest is a fully built Nigerian SMB lead generation tool. It scrapes Google Places for businesses, visits their websites with Playwright to extract contact details, scores website quality (low score = strong lead), stores everything in SQLite, and sends cold outreach emails via SMTP. All three phases are complete.

## How to Run

**API backend (always start this first):**
```bash
venv/Scripts/python.exe run_api.py
```
Do NOT use `python -m uvicorn api:app` directly on Windows ‚Äî it uses SelectorEventLoop which breaks Playwright subprocess spawning. `run_api.py` sets `WindowsProactorEventLoopPolicy` before Uvicorn starts.

**Streamlit frontend (separate terminal):**
```bash
venv/Scripts/python.exe -m streamlit run streamlit_app.py
```

**CLI scraper (no API needed):**
```bash
venv/Scripts/python.exe main.py
```

**Enrichment pipeline (run after scraping to find decision-maker emails):**
```bash
venv/Scripts/python.exe -m enricher.enricher   # scrape team pages, generate candidates
venv/Scripts/python.exe verify_contacts.py      # verify candidates via mails.so
venv/Scripts/python.exe -m enricher.drafter     # generate Gemini cold email drafts
```

**Reset enrichment state (wipes contacts + drafts, re-queues businesses for enrichment):**
```bash
venv/Scripts/python.exe reset_enrichment.py
```
Businesses already emailed (status='sent' in campaigns) are protected and never touched.

**Diagnostic / dev scripts:**
```bash
venv/Scripts/python.exe test_mailsso.py   # test mails.so API connectivity
venv/Scripts/python.exe test_port.py      # check if port 25 is open
```

**Install packages (always use venv pip, never system pip):**
```bash
venv/Scripts/pip3.exe install <package>
```

**Print encoding for terminal output:**
```bash
PYTHONIOENCODING=utf-8 venv/Scripts/python.exe script.py
```

## Architecture ‚Äî How the Pieces Fit Together

### Data Flow
```
User Input (category + city)
  ‚Üí scraper/places.py        # Google Places Text Search + Place Details
  ‚Üí scraper/website.py       # Playwright: visit each site, discover contact pages
      ‚Üí scraper/extractor.py # Regex extraction: email, phone, WhatsApp, social links
      ‚Üí scraper/scorer.py    # Score 0‚Äì100 (lower = weaker site = stronger lead)
  ‚Üí db/database.py           # Dedup against SQLite, save new leads
  ‚Üí main.py                  # Export timestamped Excel + update master Excel
```

### Two Entry Points, Same Core
`main.py` (CLI) and `api.py` (FastAPI) both call the same scraper functions. `api.py` wraps all blocking calls in `asyncio.to_thread()` and runs the job in a `BackgroundTasks` callback. Results stored in an in-memory `jobs: dict` keyed by UUID job_id. The Streamlit frontend polls `GET /results/{job_id}` every 3 seconds.

### Database Layer (`db/database.py`)
Five SQLite tables in `output/leadharvest.db`:
- `businesses` ‚Äî primary key is `email` (lowercase); `place_id` is a secondary UNIQUE constraint. A business without email is saved only if it has a place_id.
- `campaigns` ‚Äî log of every email send attempt (email, business_name, sent_at, status).
- `contacts` ‚Äî decision-maker candidate emails generated by the enricher. One row per (person, pattern) combo. Has `smtp_status` field (unknown/verified/rejected/catch_all).
- `verification_results` ‚Äî output of `verify_contacts.py`. One row per verified email with status and verified_at timestamp.
- `drafts` ‚Äî Gemini-generated cold email drafts. One row per contact, status='pending'/'sent'.

Dedup happens at two points: `get_existing_website_urls()` is passed to `places.py` so the Places API skips already-known sites. `filter_new_businesses()` is called after scraping to separate new leads from known ones before saving or exporting.

### Enrichment Pipeline (`enricher/`)
Runs after scraping to find personal emails for decision-makers at each business.

```
enricher/enricher.py      # Phase 1: scrape team pages ‚Üí generate candidate emails ‚Üí save to contacts
enricher/team_scraper.py  # Playwright: finds /team, /about, /leadership pages; Gemini extracts names+titles
enricher/email_pattern.py # Generates all pattern variants (first, first.last, f.last, firstlast, flast, lastf)
                          # If a personal email is already known, detects the domain's pattern and uses only that
verify_contacts.py        # Phase 2: probes each candidate via mails.so API; domain-level caching saves credits
                          # Strategy: probe domain with first variant ‚Üí if catch_all stamp all, if verified stop,
                          # if rejected try remaining variants
enricher/drafter.py       # Phase 3: fetches homepage text, Gemini writes personalised cold email draft per contact
```

Verification status flow: `unknown` ‚Üí `verified` / `rejected` / `catch_all` (stored in both `contacts` and `verification_results`).

`isv_mx: False` from mails.so means Google Workspace blocked the SMTP probe ‚Äî results on such domains score 40/100 and are unreliable. Don't treat "rejected" as a hard bounce if `isv_mx` is False.

### Contact Extraction Priority Chain (`scraper/website.py`)
1. Homepage HTML
2. Discovered internal contact pages (dynamic link scoring, not hardcoded paths)
3. Fallback paths: /contact, /about, /contact-us
4. If still no email and social URLs were found: visit Facebook /about, then Instagram, then Twitter (Phase 3 social fallback)

`email_source` field records where the email came from: `"website"`, `"facebook"`, `"instagram"`, `"twitter"`, or `""`.

### No-Website Businesses
Businesses with no website URL from Google Places are the strongest leads. They get `has_website=False`, `website_quality_score=0`, `"No website"` in issues, and `is_high_priority()` returns `True` unconditionally.

### Concurrency Model
`scrape_all_websites()` launches one Chromium instance shared across all sites. Each site gets an isolated browser context (fresh cookies, random user agent). A semaphore limits concurrent workers (default 5, configurable via `SCRAPE_WORKERS` in `.env`).

### Email Campaigns (`emailer/`, `db/database.py`, Streamlit tab 2)
`get_leads_for_campaign(limit)` returns leads that have an email and have never been sent to (LEFT JOIN on campaigns table). `send_email()` sends via Namecheap SMTP, then `save_campaign_send()` logs the result. Daily limit enforced by counting today's `sent` rows in campaigns.

## Key Design Decisions to Preserve

- **No emoji in `print()` calls** ‚Äî Windows cp1252 terminal crashes. Use `[TAG]` style labels.
- **`run_api.py` not `uvicorn` directly** ‚Äî ProactorEventLoop fix is mandatory on Windows.
- **DB path is `output/leadharvest.db`** ‚Äî inside project folder, satisfies IT security rules.
- **Cloudflare email decoding** in `extractor.py` ‚Äî XOR-decodes `data-cfemail` hex attributes.
- **Master Excel** (`output/exports/leadharvest_master.xlsx`) grows across runs; per-run files are separate timestamped exports. Dedup in master uses place_id first, email second.
- **`@st.cache_data(ttl=300)`** on `_fetch_categories()` ‚Äî reduces API calls from Streamlit.
- **Export bytes cached in session_state** ‚Äî prevents re-fetching Excel on every filter interaction.
